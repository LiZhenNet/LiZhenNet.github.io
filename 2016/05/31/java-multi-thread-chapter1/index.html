Java多线程编程--基础知识总结

Java
  • Java
  • 基础
  • 多线程

进程与线程

进程：系统进行资源分配和调度的独立单位。
线程：在进程中独立运行的子任务。

实现多线程

实现多线程编程的方式主要有两种:一种是继承Thread类，另一种是实现Runnable接口。Thread类实现了Runnable接口。他们之间是多态关系，
这两种创建线程在工作时的性质是一样的，没有本质的区别。

继承Thread类

看结果 MyThread的run方法执行的比较晚，也就是说当使用多线程技术时，代码运行结果与代码执行顺序或者调用顺序无关。
执行start()方法的顺序不带吗线程启动的顺序。

实现Runnable接口(注意是Runn)

多线程共享数据

可以看到打印的结果不是我们预期的依次递减，在某些JVM中，i–的操作要分为如下3步：

  1. 取原有i值
  2. 计算i-1
  3. 对i进行赋值
    在这3个步骤中，如果有多个线程同时访问，那么一定会出现非线程安全问题。

    synchronized

在方法前面加入synchronized关键字，使多个线程在执行方法时，以排队的方式进行处理。

停止线程

在java中有以下3种方法可以终止正在运行的线程：

  1. 使用退出标志，使线程正常退出，也就是当run方法完成后线程终止。
  2. 使用stop方法强制终止线程，但是不推荐这个方法，这个方法是不安全的，他可以停止一个正在运行的线程，这可能产生不可预料的结果。
  3. 使用interrupt

interrupt

调用interrupt()方法来停止线程仅仅是在当前线程中打了停止的标记，并不是真正的停止线程。

this.interrupted() 测试当前线程是否已经中断。
如果连续两次调用该方法，则第二次调用将放回false(在第一次调用已清除了其中断状态后，第二次调用检测完中断状态前，当前线程再次中断的情况除外)

this.IsInterrupted() 测试线程是否已经中断。
判断是否中断状态，但不清楚状态标识。

stop

使用stop()释放锁将会给数据造成不一致的结果

使用return停止线程

暂停线程

suspend和resume

suspend和resume 缺点：极易造成公共的同步对象的独占，还有因为线程暂停造成数据不同步。

线程的优先级

线程优先级的继承特性

线程优先级的继承特性 比如A线程启动B线程，则B线程的优先级与A是一样的。

线程优先级的规则性

CPU会尽量将执行资源让给优先级比较高的线程。

线程优先级的随机性

优先级高的并不一定先执行完。

方法

currentThread()方法

currentThread()方法可以返回代码段正在被那个线程调用的信息。

isAlive()方法

isAlive()方法是判断当前的线程是否处于活动状态。

sleep()方法

sleep()是指定毫秒数内让当前“正在执行的线程”休眠(暂停执行)。

getId()方法

getId()方法的作用是取得线程的唯一标识。

yield()方法

yield()方法的作用是放弃当前的CPU资源，将它让给其他任务去占用CPU执行时间。
但是放弃的时间不确定，有可能刚刚放弃，马上又获得cpu时间片。

守护线程

在java线程中有两种线程，一种是用户线程，另一种是守护线程。守护线程是一种特殊的线程，当进程中不存在非守护线程时，守护线程会自动销毁。
典型的守护线程就是垃圾回收线程。

当设置setDaemon(true)后五秒钟打印“执行结束”后Thread2不会执行，当没有setDaemon(true)，Thread2作为用户线程则会一种执行。
