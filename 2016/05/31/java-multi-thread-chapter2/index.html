Java多线程编程--synchronized&volatile

Java
  • Java
  • 基础
  • 多线程

对象及变量的并发访问

synchronized

关键字 synchronized 取得的锁是对象锁，而不是把一段代码或者方法当做锁。

在方法前没有加synchronized结果是：

Begin MethodA,MethodName:a
Begin MethodB,MethodName:b
End MethodB,MethodName:b
End MethodA,MethodName:a

加synchronized结果是:

Begin MethodA,MethodName:a
End MethodA,MethodName:a
Begin MethodB,MethodName:b
End MethodB,MethodName:b

A 线程先持有object 对象的Lock锁，B线程如果在这时调用object的synchronized类型的方法则需等待，也就是同步。

可重入锁

可重入锁：自己可以再次获取自己内部的锁，当存在父子类继承关系时，子类是完全通过“可冲入锁”调用父类的同步方法。

出现异常，锁自动释放

同步不具有继承性

子类重写父类synchronized修饰的方法，当没有加synchronized关键字，则以非同步的方式运行。

将任意对象作为对象监视器

当使用synchronized同步方法或者synchronized(this)同步代码块时：

  1. 对其他使用synchronized同步方法或者synchronized(this)同步代码块调用呈阻塞状态。
  2. 同一时间只有一个线程可以执行synchronized同步方法或者synchronized(this)同步代码块中的代码。

当使用synchronized(非this对象)时：

  1. 同一时间只有一个线程可以执行synchronized(非this对象)同步代码块中的代码。

使用synchronized(非this对象)优点是：

如果有一个类中有很多synchronized方法，当执行一个的时候，这时虽然能实现同步，但是会阻塞到其他的synchronized方法。
但是如果使用沟通不代码块锁非this对象，则不会与其他锁this同步方法争抢this锁。

synchronized修饰static方法

synchronized修饰static方法跟synchronized修饰非static方法不同：

  1. synchronized修饰static方法是给class类上锁。
  2. synchronized修饰非static方法是给对象上锁。

数据类型String的常量池

在jvm中具有String常量池缓存的功能，因此synchronized()同步代码块不与String联合使用。

变量在内存中工作的过程

  1. read和load阶段，从主存复制变量到当前线程的工作内存。
  2. use和assign阶段，执行代码，改变共享变量的值。
  3. store和write阶段，用工作内存数据刷新主存对应变量的值。
