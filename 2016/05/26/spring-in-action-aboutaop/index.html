Spring In Action 笔记 --关于AOP(未完成，待研究深入)

AOP 术语

  1. 连接点（Joinpoint）连接点就是应用程序执行过程中切面插入的地方。AOP中表示为“在哪里干“
  2. 切入点（Pointcut）切入点定义了通知在那些连接点。 在AOP中表示为“在哪里干”的集合
  3. 通知（Advice） 通知切面的实际实现。在AOP中表示”干什么“
  4. 切面（Aspect） 切面就是你想要实现的交叉功能，比如日志记录切面。在AOP中表示为“在哪干”和“干什么”集合”
  5. 引入(Introduction) 引入允许你为已存在的类添加新方法和属性。 在AOP中表示为“引入什么“
  6. 目标对象(Target) 目标对象是被通知的对象。在AOP中表示为“对谁干”
  7. 织入（Weaving）织入是将切面应用到目标对象而创建一个新的代理对象的过程。

    织入发生在目标对象生命周期的多个点上：
    编译期 –切面在目标对象编译时织入，这个需要一个特殊的编译器。
    类装载期 –切面在目标对象被载入到JVM中时织入，这个需要一个特殊的类载入器， 它在类载入到应用系统之前增强目标对象的字节码。
    运行期 –漆面在应用系统运行时织入，通常AOP容器将在织入切面的时候动态生成委托目标对象的代理对象。

  8. 代理（Proxy）代理是将通知应用到目标对象后创建的对象。

Spring 通知类型

  1. Around 接口org.aopalliance.intercept.MethodInterceptor 拦截对目标对象的调用
  2. Before 接口org.springframework.aop.BeforeAdvice 在目标方法被调用前调用
  3. After 接口org.springframework.aop.AfterReturningAdvice 在目标方法被调用后调用
  4. Throws 接口org.springframework.aop.ThrowsAdvice 当目标函数抛出异常时调用

Spring 定义切入点

核心接口是

  1. ClassFilter：通过类来过滤切面，实现这个接口的类决定了以参数传进来的类是否应该被通知。实现这个接口的类一般根据类名来判断。
  2. MethodMatcher：通过方法来过滤，先执行matches(Method m,Class targetClass)，如果返回true，然后通过IsRuntime来判断MethodMatcher是
    静态还是动态，如果是动态的调用 matches(Method m, Class target, Object[] args)根据运行时的值来决定通知是否需要被应用，也就是说如果
    是动态的每次都会调用matches(Method m, Class target, Object[] args)，所以首选静态切入点。
    matches(Method m,Class targetClass) 和isRuntime 只是在代理类创建时调用一次。

理解 Advisor

Advisor是spring通知和切入点组合到一起的一个对象
