Java多线程编程--线程间通信

Java
  • Java
  • 基础
  • 多线程

线程间通信

等待/通知机制

方法wait()的作用是使当前执行代码的线程进行等待，wait()方法迎来将当前线程置入“预执行队列”中，并且在wait()所在代码行处停止执行，
直到接到通知或当前线程被中断为止。在调用wait()之前，线程必须获得该对象的对象级别锁，在执行wait()方法后，当前线程释放锁。
在从wait()返回前，线程与其他线程竞争重新获得锁。如果调用wait()时没有持有适当的锁，则会抛出IllegalMonitorStateException。

方法notify()也要在获取该对象的对象级别锁，如果没有持有适当的锁，则会抛出IllegalMonitorStateException。该方法用来
通知那些可能等待该对象的对象锁的其他线程，如果有多个线程等待，则由线程规划器随机挑选出其中一个呈wait状态的线程，对其发出通知
notify，并使它等待获取该对象的对象锁。

用一句话总结一下wait和notify：wait使线程停止运行，而notify使停止的线程继续运行。

结果：

开始wait
开始notify
结束notify
结束wait

wait()方法可以使用该方法的线程释放公共资源的锁，然后从运行状态退出，进入等待队列，直到被再次唤醒。
notify()方法可以随机唤醒等待队列中等待同一共享资源的“一个”线程，并使线程退出等待队列，进入可运行状态。
notifyAll()方法可以唤醒等待队列中等待同一共享资源的“全部”线程，并使线程退出等待队列，进入可运行状态。

wait()/notify()释放锁

当执行到wait()时立即释放锁，notify()必须执行完方法所在的同步synchronized代码块后才释放锁。

join方法

方法join的作用是使所属线程对象x正常执行run()方法中的人物，而使当前线程z进行无限期的阻塞，等待线程x销毁后
再继续执行线程z后面的代码。

方法join(long)与sleep(long)的区别

方法join(long)的功能在内部是使用wait(long)方法来实现，所以join(long)方法具有释放锁的特点。

方法join()后面的代码提前执行

运行结果为:

Begin A
End A
end
Begin B
End B

这可能跟我们预料的结果不一样，我们以为join(2000) 会阻塞，end 应该在 End B 后面打印输出。但事实并不是这样，造成这一原因是：

  1. b.join(2000) 先抢到B锁，然后将B锁释放
  2. ThreadA 抢到锁，执行完 释放锁，这时已经过去了5秒
  3. 这时join(2000)和ThreadB争抢锁，join(2000)再次获得锁，发现时间已过，释放锁后打印 end
  4. ThreadB抢到锁并执行完。

ThreadLocal

类ThreadLocal主要解决的就是每个线程绑定自己的值,ThreadLocal类比喻成全局存放数据的盒子，盒子中可以存储每个线程的私有数据。

InheritableThreadLocal

类 InheritableThreadLocal 可以在子线程中取得父线程继承下来的值
