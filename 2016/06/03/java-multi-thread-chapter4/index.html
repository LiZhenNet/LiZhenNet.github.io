Java多线程编程--Lock的使用

Java
  • Java
  • 基础
  • 多线程

Lock的使用

使用ReentrantLock类

在java多线程中，可以使用synchronized关键字来实现线程之间同步互斥，但是jdk1.5 中新增加了ReentrantLock类也能达到同样的效果，
并在扩展功能上更强大，比如具有嗅探锁定，多路分支通知等功能，而且在使用上也更加灵活。

使用Condition实现等待/通知

结果如下：

begin awaitB
begin awaitA
begin signalAllA
end signalAllA
end awaitA

可以看到Condition可以实现通知指定部分线程

公平锁与非公平锁

公平锁表示线程获取锁的顺序是按照线程加锁的顺序来分配的,既先来先得的FIFO先进先出顺序.
非公平锁就是一种获取锁的抢占机制,是随机获得锁的

可以看到启动的顺序和获得锁的顺序是一致的。

常用方法

  1. getHoldCount() 查询当前线程保持此锁定的个数，也就是调用lock（）方法的次数。
  2. getQueueLength() 返回正等待获取此锁定的线程估计数。

  3. getWaitQueueLength(Condition condition) 作用是返回等待此锁定相关的给定条件Condition的线程估计数。

  4. hasQueuedThread(Thread thread) 查询指定的线程是否正在等待获取此锁定。

  5. hasQueuedThreads() 是否有线程正在等待获取此锁定。

  6. hasWaiters(Condition condition)作用是查询是否有线程正在等待与锁定有关condition条件。

  7. isFair() 判断是不是公平锁。

  8. isHeldByCurrentThread() 的所用是查询当前线程是否保持此锁定。
  9. isLocked()的所用是查询此锁定是否由任意线程保持。
  10. tryLock() 尝试获得锁

ReentrantReadWriteLock类

ReentrantReadWriteLock 表示有两个锁，一个是读操作相关的锁，也称为共享锁，另一个是写操作的锁，也叫排他锁。

共享锁

结果为:

Thread-0获得读锁 1464935237097
Thread-1获得读锁 1464935237097

排他锁

结果为:

Thread-0获得读锁 1464937326234
Thread-1获得读锁 1464937336235

“读写” “写读” “写写” 互斥 ,读读共享锁 不互斥
